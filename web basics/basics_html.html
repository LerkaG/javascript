Ссылки

У ссылок есть возможность совершать звонки на телефоны, отправлять сообщения или звонить по скайпу:
  • Google.com (a href="http://google.com" target="_blank" rel="noreferrer noopener" title="Поисковая система Google")
  • +1 (425) 123-45-63 (a href="tel:+14251234563")
  • example@mail.ru (a href="mailto:example@mail.ru")
  • Call on Skype (a href="skype:имя-пользователя?call")
  • Chat on Skype (a href="skype:имя-пользователя?chat")
  • Add to Skype (a href="skype:имя-пользователя?add")
  • Send file on Skype (a href="skype:имя-пользователя?sendfile")
  • Go to Section 2 (a href="#section2") - ссылка на другой раздел на той же странице
  • Execute JavaScript (a href="javascript:alert('Hello World!');) - ссылка на JavaScript
href - указывает адрес на ту страницу, на которую перейдет пользователь при нажатии на текст ссылки
target - указывает то, в каком окне откроется страница при клике на ссылку
_blank - значение target страница открывается в новой вкладке браузера
rel="noreferrer noopener" - открывать страницу на новой вкладке всегда с этим
download="путь к файлу" - сообщает браузеру, что указанный ресурс должен быть загружен в момент, когда пользователь щелкает по ссылке.
title - добавляет всплывающую подсказку к тексту ссылки

Для составления пути относительных ссылок есть несколько специальных обозначений.
/ - указывает на корневую папку, например /img/smile.png
./ - указывает на текущую папку, например ./img/smile.png
../ - подняться на одну папку выше, например ../img/smile.png

Основные теги

article - независимая, отделяемая, и имеющая значение вне, часть документа, например твит, статья, виджет с рекламой. Желателен заголовок.
section - смысловой раздел документа. Неотделяемый от основного документа. Желателен заголовок.
nav - навигационный раздел со ссылками на другие страницы или другие части страниц. Используется для основной навигации, а не для всех групп ссылок в документе.
header - вводная часть смыслового раздела или всего сайта. Часто используется как шапка всей страницы или сложной секции. Может быть несколько на странице.
main - основное, не повторяющееся на других страницах, содержание. Может быть только один на странице.
footer - финальная часть раздела или всего сайта, часто содержит копирайт, список ссылок на социальные сети и так далее. Может быть несколько на странице.
h1-h6 - используются только для заголовков. h1 - один раз на странице. h2-h6 могут повторятся. Должны идти по порядку от h1-h6
ol - создает нумерованный список, т.е. каждый элемент списка начинается с числа.
ul - создает маркированный список, каждый элемент которого начинается с небольшого символа — маркера.
li - определяет отдельный элемент списка.

script - Тег используется для встраивания на стороне клиента сценария JavaScript. Элемент либо содержит утверждения сценариев, или он указывает на внешний файл сценария через атрибут Src.
Обычное использование JavaScript - это манипуляции с изображениями, проверка формы и динамические изменения содержимого.
Существует несколько способов выполнения внешнего скрипта:
Если async = "async": сценарий выполняется асинхронно с остальной частью страницы (сценарий будет выполняться, пока страница продолжает анализ)
Если async отсутствует и defer = "defer": скрипт выполняется после завершения анализа страницы
Если нет ни асинхронного, ни отсроченного: скрипт выбирается и выполняется немедленно, прежде чем браузер продолжит анализ страницы
Также обратите внимание на элемент noscript для пользователей, которые отключили сценарии в своем браузере или имеют браузер, который не поддерживает сценарии на стороне клиента.
АТРИБУТЫ: async="async", charset="charset", defer="defer", src="URL", type="media_type"
ПРИМЕР: Testing
