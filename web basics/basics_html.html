Ссылки

У ссылок есть возможность совершать звонки на телефоны, отправлять сообщения или звонить по скайпу:
  • Google.com (a href="http://google.com" target="_blank" rel="noreferrer noopener" title="Поисковая система Google")
  • +1 (425) 123-45-63 (a href="tel:+14251234563")
  • example@mail.ru (a href="mailto:example@mail.ru")
  • Call on Skype (a href="skype:имя-пользователя?call")
  • Chat on Skype (a href="skype:имя-пользователя?chat")
  • Add to Skype (a href="skype:имя-пользователя?add")
  • Send file on Skype (a href="skype:имя-пользователя?sendfile")
  • Go to Section 2 (a href="#section2") - ссылка на другой раздел на той же странице
  • Execute JavaScript (a href="javascript:alert('Hello World!');) - ссылка на JavaScript
href - указывает адрес на ту страницу, на которую перейдет пользователь при нажатии на текст ссылки
target - указывает то, в каком окне откроется страница при клике на ссылку
_blank - значение target страница открывается в новой вкладке браузера
rel="noreferrer noopener" - открывать страницу на новой вкладке всегда с этим
download="путь к файлу" - сообщает браузеру, что указанный ресурс должен быть загружен в момент, когда пользователь щелкает по ссылке.
title - добавляет всплывающую подсказку к тексту ссылки

Для составления пути относительных ссылок есть несколько специальных обозначений.
/ - указывает на корневую папку, например /img/smile.png
./ - указывает на текущую папку, например ./img/smile.png
../ - подняться на одну папку выше, например ../img/smile.png

Основные теги

article - независимая, отделяемая, и имеющая значение вне, часть документа, например твит, статья, виджет с рекламой. Желателен заголовок.
section - смысловой раздел документа. Неотделяемый от основного документа. Желателен заголовок.
nav - навигационный раздел со ссылками на другие страницы или другие части страниц. Используется для основной навигации, а не для всех групп ссылок в документе.
header - вводная часть смыслового раздела или всего сайта. Часто используется как шапка всей страницы или сложной секции. Может быть несколько на странице.
main - основное, не повторяющееся на других страницах, содержание. Может быть только один на странице.
footer - финальная часть раздела или всего сайта, часто содержит копирайт, список ссылок на социальные сети и так далее. Может быть несколько на странице.
h1-h6 - используются только для заголовков. h1 - один раз на странице. h2-h6 могут повторятся. Должны идти по порядку от h1-h6
ol - создает нумерованный список, т.е. каждый элемент списка начинается с числа.
ul - создает маркированный список, каждый элемент которого начинается с небольшого символа — маркера.
li - определяет отдельный элемент списка.

HTML теги по алфавиту

a - Тэг определяет гиперссылку, которая используется для связи с одной страницы на другую.
АТРИБУТЫ:
download="filename", href="URL", hreflang="language_code", media="media_query", ping="list_of_URLs",
referrerpolicy="no-referrer", "no-referrer-when-downgrade", "origin", "origin-when-cross-origin", "unsafe-url",
rel="alternate", "author", "bookmark", "external", "help", "license", "next", "nofollow", "noreferrer", "noopener", "prev", "search", "tag",
target="_blank", "_parent", "_self", "_top", "framename", type-"media_type".

abbr - Тэг определяет аббревиатуру или аббревиатуру, как «HTML», «CSS», «мистер», «Доктор», «как можно скорее», «ATM».

address - Тэг определяет контактную информацию автора / владельца документа или статьи о.

area - Тег определяет область внутри изображения-карты map (изображение-карта представляет собой изображение с интерактивными областями).
АТРИБУТЫ:
alt="text", coords="coordinates", download="filename", href="URL", hreflang="language_code", media="media query",
rel="alternate", "author", "bookmark", "help", "license", "next", "nofollow", "noreferrer", "prefetch", "prev", "search", "tag",
shape="default", "rect", "circle", "poly", target="_blank", "_parent", "_self", "_top", "framename", type="media_type"

article - Тег определяет независимое, самодостаточное содержание. Потенциальные источники для article элемента: Пост форума, Сообщение блога, Новость

aside - Тэг определяет некоторый контент в стороне от содержания его помещают в. Дополнительный контент должен быть косвенно связан с окружающим контентом.
aside Содержимое часто размещается в документе как боковая панель.

audio - Тег используется для встраивания контента звука в документе. audioТег содержит один или несколько source тегов с различными источниками звука.
Браузер выберет первый источник, который он поддерживает. Текст между audio тегами будет отображаться только в браузерах, не поддерживающих элемент <аудио>.
АТРИБУТЫ:
autoplay="autoplay", controls="controls", loop="loop", muted="muted", preload="auto", "metadata", "none", src="URL"

base - Тег определяет базовый URL и / или цель для всех относительных URL - адресов в документе. Тег должен иметь либо HREF или целевой атрибут настоящего, или оба.
В документе может быть только один base элемент, и он должен быть внутри элемента .
АТРИБУТЫ: href="URL", target="_blank", "_parent", "_self", "_top"

bdi - тег изолирует часть текста, которая может быть отформатирована в другом направлении, чем другой текст за ее пределами. Полезен при встраивании пользовательского контента с неизвестным направлением текста.
ПРИМЕР:
User إيان: 90 points
User إيان<: 90 points

bdo - Тег используется для переопределения текущего направления текста.
АТРИБУТЫ: dir="ltr", "rtl"

blockquote - Тег определяет раздел, который процитированный из другого источника. Браузеры обычно делают отступы для элементов. Использовать q для встроенных (коротких) цитат.
АТРИБУТЫ: cite="URL"
ПРИМЕР:
For 50 years, WWF has been protecting the future of nature.

button - Тег определяет кнопку интерактивной. Внутри элемента можно поместить текст (и такие теги, как i, strong, br, img и т. Д.).
